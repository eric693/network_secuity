Linux 是什麼？為何學 Linux？

基礎概念 預計 15 分鐘

什麼是 Linux？

Linux 是一套開源的、類 Unix 的作業系統核心（Kernel），由 Linus Torvalds 於 1991 年開始開發。搭配 GNU 工具集後形成完整的作業系統，通常稱為 GNU/Linux。

Linux 以「發行版（Distribution）」的形式提供給用戶，常見的發行版包括 CentOS、Ubuntu、Debian、Kali Linux 等。在資安領域，CentOS/RHEL 是企業伺服器的主流環境，Kali Linux 則是滲透測試的標準平台。

核心 (Kernel)
作業系統的核心程式，負責管理硬體資源（CPU、記憶體、磁碟）、進程調度、檔案系統、設備驅動等。
Shell
用戶與核心之間的介面，接收用戶輸入的命令並傳達給核心執行。常見的 Shell：bash、sh、zsh、fish。
發行版 (Distro)
Linux Kernel + GNU 工具 + 套件管理器 + 默認配置 的組合。CentOS 使用 yum/dnf，Ubuntu 使用 apt。
開源 (Open Source)
原始碼完全公開，任何人都可以查看、修改、重新分發。這使得 Linux 的安全問題能被全球社群快速發現並修補。

為什麼資安工程師必須會 Linux？

根據統計，全球 96% 的 Web 伺服器運行 Linux，幾乎所有的雲端基礎設施（AWS EC2、GCP、Azure）默認也是 Linux。更重要的是：

應用場景Linux 的角色具體例子
滲透測試 攻擊平台 Kali Linux 上運行 Metasploit、Nmap、Burp Suite
目標系統 被滲透的對象 CentOS/Ubuntu Web 伺服器、資料庫伺服器
應急響應 事件分析平台 日誌分析、進程排查、惡意文件分析
靶場搭建 實驗環境 用 Docker 在 Linux 上快速部署 Vulhub 漏洞環境
安全開發 開發與部署環境 Python 安全腳本、自動化工具的運行環境
學習重點

本階段的目標不是成為 Linux 系統管理員，而是掌握資安工作中每天會用到的 Linux 操作技能：文件操作、權限管理、日誌分析、進程查看、網路配置和腳本基礎。

正確！Linux 的超級管理員帳號固定為 root，其 UID（User ID）為 0。root 可以執行任何操作，包括修改系統文件、刪除所有資料、安裝軟體等。在滲透測試中，提升到 root 權限（Root Privilege Escalation）是最終目標之一。在日常操作中，應避免直接使用 root，改用 sudo 執行需要特權的命令。

VMware 安裝與 Linux 系統安裝

在正式學習 Linux 之前，需要先搭建實驗環境。推薦使用 VMware Workstation 建立虛擬機，這樣可以在不影響宿主機的情況下自由實驗，犯錯也可以恢復快照。

為什麼選擇虛擬機？

快照 (Snapshot)
保存當前虛擬機的完整狀態，出錯後可以一鍵還原到之前的狀態。這在學習和實驗時非常安全。
隔離性
虛擬機在沙箱環境中運行，即使虛擬機系統被攻陷或出錯，宿主機也不受影響。滲透測試靶機必須用虛擬機。
網路模擬
VMware 可以模擬複雜的網路拓撲（NAT、橋接、僅主機），方便搭建攻防實驗環境。
克隆 (Clone)
從一個配置好的虛擬機快速複製出多台，節省重複安裝的時間。搭建多機攻防環境時非常有用。

注意事項

在分析惡意軟體、搭建漏洞靶機時，務必將虛擬機網路設為「僅主機模式」，防止惡意程式透過虛擬機感染宿主機或外部網路。

小測驗
你要在虛擬機上搭建一個存在嚴重漏洞的 Web 應用靶機進行練習，應該選擇哪種 VMware 網路模式最安全？
A橋接模式，方便從任意設備訪問靶機
BNAT 模式，虛擬機可以上網更新靶機
C僅主機模式，完全隔離防止漏洞被外部利用
D不需要考慮網路模式，靶機不重要
正確！搭建漏洞靶機時應使用「僅主機模式」。橋接模式下，漏洞可能被同一局域網的其他設備利用；NAT 模式雖較安全但靶機仍可向外發起連接。僅主機模式完全隔離了外部網路，只允許宿主機與虛擬機之間通信，是最安全的選擇。

目錄導航：pwd / cd / tree / mkdir

Linux 的一切都是文件，包括目錄（目錄也是一種特殊的文件）。掌握目錄導航命令是使用 Linux 的第一步。

pwd
pwd
Print Working Directory，顯示當前所在的完整絕對路徑。迷路時最有用的命令。
cd
cd [目錄路徑]
Change Directory，切換工作目錄。cd ~ 回 home 目錄，cd - 回上次的目錄，cd .. 回上一層。
tree
tree [選項] [目錄]
以樹狀結構顯示目錄內容。tree -L 2 只顯示 2 層深度，tree -a 顯示隱藏文件。需安裝：yum install tree。
mkdir
mkdir [選項] 目錄名
Make Directory，創建目錄。mkdir -p a/b/c 可遞迴創建多層目錄，mkdir -m 755 dir 同時設定權限。
實戰演練 — 目錄導航
# 查看當前位置 [root@localhost ~]# pwd /root # 切換到 /tmp 目錄 [root@localhost ~]# cd /tmp [root@localhost tmp]# pwd /tmp # 創建多層目錄結構 [root@localhost tmp]# mkdir -p project/src/main # 查看目錄樹狀結構 [root@localhost tmp]# tree project project └── src └── main 2 directories, 0 files # 使用 ~ 快速回到 root 的 home 目錄 [root@localhost tmp]# cd ~ [root@localhost ~]# pwd /root # cd - 回到剛才去過的目錄 [root@localhost ~]# cd - /tmp

應急響應知識點

在進行 Linux 入侵排查時，ls -la --full-time /tmp、ls -la /var/www/html 等命令可以快速找到近期被創建或修改的可疑文件。攻擊者入侵後常在 /tmp 目錄放置 Webshell 或惡意腳本，因為 /tmp 對所有用戶可寫。

cp — 複製文件與目錄

cp（copy）命令用於複製文件或目錄，是日常操作和備份中最常用的命令之一。

cp 命令使用範例
# 基本複製（文件到文件） [root@localhost ~]# cp source.txt destination.txt # 複製文件到目錄 [root@localhost ~]# cp notes.txt /tmp/ # 遞迴複製整個目錄（-r 或 -R） [root@localhost ~]# cp -r /etc/nginx /tmp/nginx_backup # 複製並保留原始文件屬性（時間戳、權限、所有者） [root@localhost ~]# cp -p important.conf /backup/ # 複製前詢問是否覆蓋（-i = interactive） [root@localhost ~]# cp -i file.txt /tmp/ cp: overwrite '/tmp/file.txt'? y # 常用組合：-rp 遞迴且保留屬性，備份常用 [root@localhost ~]# cp -rp /var/www/html /var/www/html_backup_20250101
實戰應用：修改前先備份

在修改任何重要的配置文件（如 /etc/ssh/sshd_config、/etc/nginx/nginx.conf）之前，務必先備份：cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak。這是一個非常好的習慣，出錯時可以快速恢復。

rm 刪除 / mv 移動與重命名

極度危險

Linux 的 rm 命令沒有回收站！刪除後資料直接消失，幾乎無法恢復。特別是 rm -rf / 這個命令（刪除根目錄下所有文件）是毀滅性的，絕對不能執行。操作前請三思，確認路徑無誤再按 Enter。

rm / mv 命令使用範例
===== rm 刪除命令 ===== # 刪除單個文件 [root@localhost ~]# rm test.txt # 刪除前詢問確認（推薦） [root@localhost ~]# rm -i important.txt rm: remove regular file 'important.txt'? y # 強制刪除不詢問（小心！） [root@localhost ~]# rm -f file.txt # 遞迴刪除整個目錄（危險！確保路徑正確） [root@localhost ~]# rm -rf /tmp/test_dir ===== mv 移動/重命名命令 ===== # 重命名文件 [root@localhost ~]# mv old_name.txt new_name.txt # 移動文件到其他目錄 [root@localhost ~]# mv notes.txt /tmp/ # 移動整個目錄 [root@localhost ~]# mv /tmp/old_dir /var/www/new_dir # 移動前詢問是否覆蓋 [root@localhost ~]# mv -i file.txt /tmp/
小測驗
以下哪個命令可以將 /home/user/documents 目錄下的所有文件複製到 /backup/ 目錄，並保留原始文件的時間戳和權限？
Acp /home/user/documents /backup/
Bcp -rp /home/user/documents /backup/
Cmv /home/user/documents /backup/
Dcp -f /home/user/documents /backup/
正確！-r (recursive) 讓 cp 遞迴複製整個目錄，-p (preserve) 保留文件的時間戳、權限和所有者。選項 A 沒有 -r 無法複製目錄，選項 C 是移動（原始資料會消失），選項 D 的 -f 是強制覆蓋目標但不保留屬性。

VIM 編輯器介紹

VIM（Vi IMproved）是 Linux 上最重要的文字編輯器，也是讓初學者最頭疼的程式之一。但一旦掌握，效率極高，而且在沒有圖形介面的伺服器環境中，VIM 幾乎是唯一的選擇。

VIM 的核心設計理念是：手不離鍵盤，透過不同模式和快捷鍵完成所有編輯操作，無需觸碰滑鼠。

初學者常見問題

「我打開了 VIM 但不知道怎麼退出！」— 這幾乎是所有 Linux 新手的第一個問題。記住：按 Esc 確保回到普通模式，然後輸入 :q! 強制退出（不儲存），或 :wq 儲存後退出。

VIM 的三大模式

普通模式 (Normal Mode)
VIM 啟動後默認進入此模式。此模式下輸入的字母都是命令，不能直接輸入文字。按 Esc 從任何模式回到此模式。
插入模式 (Insert Mode)
類似一般的文字編輯器，可以直接輸入和修改文字。按 i（在游標前插入）、a（在游標後插入）、o（在下方新增一行）進入。
命令行模式 (Command Mode)
在普通模式下按 : 進入，可以執行儲存、退出、搜尋替換等命令。如 :w 儲存、:q 退出、:set nu 顯示行號。
可視模式 (Visual Mode)
在普通模式下按 v 進入，可以選取文字塊進行複製、刪除等操作。V 是行選取，Ctrl+V 是列選取。

VIM 基本操作實戰

VIM 完整操作流程
# 1. 開啟/創建文件 [root@localhost ~]# vim test.conf # 進入 VIM 後的操作（在 VIM 內部）： # === 進入插入模式 === i → 在游標位置之前插入 a → 在游標位置之後插入 o → 在當前行下方新增一行並進入插入模式 I → 移到行首並進入插入模式 A → 移到行尾並進入插入模式 O → 在當前行上方新增一行 # === 回到普通模式 === Esc → 從任何模式回到普通模式 # === 儲存與退出（命令行模式）=== :w → 儲存（Write） :q → 退出（Quit，前提是無未儲存修改） :wq → 儲存並退出（最常用！） :q! → 強制退出，不儲存任何修改 :wq! → 強制儲存並退出（唯讀文件時使用） :x → 等同 :wq（有修改時才寫入） # === 在命令行模式下的實用命令 === :set nu → 顯示行號 :set nonu → 關閉行號 :set paste → 貼上模式（避免自動縮排） :/keyword → 向下搜尋 keyword :?keyword → 向上搜尋 keyword :s/old/new/g → 在當前行替換 old 為 new :%s/old/new/g → 全文替換
資安實戰：修改 SSH 配置

在加固伺服器時，你會頻繁用 VIM 修改配置文件：
vim /etc/ssh/sshd_config
找到 PermitRootLogin yes，改為 PermitRootLogin no，
儲存後重啟 SSH：systemctl restart sshd。
這一步驟可以防止攻擊者直接暴力破解 root 帳號的 SSH 登入。

相對路徑與絕對路徑

在 Linux 中，指定文件位置有兩種方式：絕對路徑和相對路徑。理解這兩者的差異是避免操作失誤的關鍵。

絕對路徑 (Absolute Path)
從根目錄 / 開始的完整路徑，不依賴當前目錄。例如：/etc/nginx/nginx.conf、/home/user/documents/report.txt。
相對路徑 (Relative Path)
相對於當前工作目錄的路徑。. 代表當前目錄，.. 代表上一層目錄。例如：在 /home/user 時，./documents/report.txt 等於 /home/user/documents/report.txt。
路徑使用範例
# 當前位置：/var/www [root@localhost www]# pwd /var/www # 相對路徑：進入 html 子目錄 [root@localhost www]# cd html # 等同於：cd /var/www/html # 相對路徑：回到上一層 var 目錄 [root@localhost html]# cd ../.. # 等同於：cd / # 絕對路徑（從任何位置都有效） [root@localhost /]# cat /etc/hostname web-server-01 # 快速識別：以 / 開頭的是絕對路徑 /etc/passwd ← 絕對路徑 ./config.txt ← 相對路徑（當前目錄） ../logs/error.log ← 相對路徑（上層的 logs 目錄）

小測驗
你需要從 /var/log/nginx/access.log 中找出所有狀態碼為 404 的請求，並統計總數。以下哪個命令組合是正確的？
Afind "404" /var/log/nginx/access.log | count
Bgrep " 404 " /var/log/nginx/access.log | wc -l
Ccat /var/log/nginx/access.log | find 404
Dtail /var/log/nginx/access.log | grep 404
正確！grep " 404 " file 搜尋包含「 404 」（注意空格）的行，管道給 wc -l（word count 的行數模式）統計行數。選項 A 語法錯誤，選項 C 的 cat+find 不是正確用法，選項 D 的 tail 只顯示尾部行，不是全部。

資安提權知識點

tar 是 Linux 中最常用的打包工具，可以結合 gzip、bzip2 等工具進行壓縮。

# ===== 打包並壓縮 ===== # czf = Create（創建）+ z（gzip 壓縮）+ f（指定文件名） [root@localhost ~]# tar -czf backup.tar.gz /var/www/html # 生成 backup.tar.gz # ===== 查看壓縮包內容（不解壓）===== [root@localhost ~]# tar -tzf backup.tar.gz # ===== 解壓縮 ===== # xzf = eXtract（解壓）+ z（gzip）+ f（指定文件名） [root@localhost ~]# tar -xzf backup.tar.gz # 解壓到當前目錄 # 解壓到指定目錄 [root@localhost ~]# tar -xzf backup.tar.gz -C /tmp/restore # bzip2 壓縮（壓縮比更高，但更慢） [root@localhost ~]# tar -cjf archive.tar.bz2 /data # 其他常見格式 [root@localhost ~]# zip -r archive.zip /data [root@localhost ~]# unzip archive.zip -d /tmp/
記憶技巧

tar 的參數邏輯：c（創建）/ x（解壓）/ t（查看） + z（gzip/.gz）/ j（bzip2/.bz2） + f（指定文件名，必須放最後）。壓縮用 czf，解壓用 xzf，一旦記住這個規律就不會忘了。

Linux 權限配置詳解

Linux 的權限系統是其安全模型的核心。理解和正確配置權限，是防止未授權訪問的第一道防線，也是進行提權攻擊分析的基礎。

權限的組成

文件權限字串結構：-rwxr-xr--

chmod 與 chown 權限管理命令
===== chmod：修改文件權限 ===== # 數字方式：rwxr-xr-- = 754 [root@localhost ~]# chmod 755 script.sh # 755 = rwxr-xr-x（擁有者全權，群組/其他可讀執行） [root@localhost ~]# chmod 644 config.txt # 644 = rw-r--r--（擁有者可讀寫，其他只讀） [root@localhost ~]# chmod 600 ~/.ssh/id_rsa # 600 = rw-------（只有擁有者可讀寫，SSH 私鑰必須這樣設） [root@localhost ~]# chmod -R 755 /var/www/html # -R 遞迴設定整個目錄 # 符號方式：更直觀 [root@localhost ~]# chmod u+x script.sh # 給擁有者加執行權限 [root@localhost ~]# chmod o-r secret.txt # 去掉其他人的讀取權限 ===== chown：修改文件擁有者 ===== [root@localhost ~]# chown nginx:nginx /var/www/html # 將 /var/www/html 的擁有者改為 nginx 用戶和群組 [root@localhost ~]# chown -R www-data:www-data /var/www
危險的 777 權限

chmod 777 /var/www/html 表示任何用戶都可以讀、寫、執行此目錄下的所有文件！這是非常嚴重的安全隱患。如果 Web 目錄是 777，攻擊者一旦找到文件上傳漏洞，就能直接上傳 Webshell 並執行。Web 目錄應設為 755（目錄）和 644（文件），且擁有者為 nginx/www-data，而非 root。

小測驗
一個文件的權限為 -rwxr-x---，請問這個文件的八進制權限數字是什麼？

正確！拆解：rwx=7，r-x=5，---=0，合起來是 750。Owner（rwx）= r(4)+w(2)+x(1) = 7，Group（r-x）= r(4)+w(0)+x(1) = 5，Others（---）= 0+0+0 = 0，所以是 750。

Docker 概念與安裝

Docker 是一個容器化平台，讓應用程式及其依賴項可以打包成「映像（Image）」，在任何環境中一致地運行。對於資安工程師，Docker 的最大價值是快速搭建漏洞靶場環境（如 Vulhub）。

映像 (Image)
只讀的模板，包含應用程式及其所有依賴項。類似虛擬機的快照。從映像創建出容器。
容器 (Container)
映像的運行實例。可以啟動、停止、刪除。容器是隔離的進程，共享宿主機的作業系統核心。
Registry
映像倉庫，默認是 Docker Hub（hub.docker.com）。Vulhub 的所有漏洞環境映像都存放在此。
Dockerfile
定義映像如何建構的腳本文件，描述基礎映像、安裝什麼軟體、複製什麼文件、執行什麼命令。

Vulhub — 最重要的靶場工具

Vulhub 是一個基於 Docker-Compose 的開源漏洞練習平台，包含數百個真實漏洞環境（Struts2、Log4j、Spring、Redis 等）。只需 2 條命令即可啟動任何漏洞環境。這將是你在 Stage 4 中進行漏洞複現練習的核心工具。

ELK 日誌收集系統介紹

ELK 是 Elasticsearch + Logstash + Kibana 三個開源工具的組合，是企業級日誌收集、分析和視覺化的主流解決方案。

Elasticsearch
分散式搜尋與分析引擎，負責儲存和搜尋日誌數據。提供強大的全文搜尋和聚合分析能力。
Logstash
數據收集和處理管道，負責從各種來源收集日誌、過濾處理，然後輸出到 Elasticsearch。
Kibana
視覺化面板，提供 Web 界面用於搜尋日誌、創建圖表和儀表板，讓日誌分析更直觀。
Filebeat
輕量級日誌收集器，部署在各伺服器上採集日誌文件，傳送給 Logstash 或直接給 Elasticsearch。
資安應用：SIEM 系統基礎

ELK 是構建企業 SIEM（安全信息與事件管理）系統的核心組件。透過 ELK，安全團隊可以：集中管理所有伺服器的安全日誌、即時搜尋可疑行為、設置告警規則（如某 IP 登入失敗超過 10 次自動告警）、進行安全事件的溯源分析。

堡壘機概念與 JumpServer 安裝

堡壘機（Jump Server / Bastion Host）是企業用於統一管理、審計伺服器訪問的安全設備。所有的運維操作必須先登入堡壘機，再由堡壘機連接到目標伺服器，實現訪問控制和操作審計。

訪問控制
統一管理哪些用戶可以訪問哪些伺服器，不需要將伺服器的帳號密碼分發給每個運維人員。
操作審計
記錄所有用戶在伺服器上執行的命令和操作，可回放，滿足合規要求（如等保2.0的審計要求）。
單點登入
用戶只需記住堡壘機的帳號密碼，透過堡壘機訪問所有授權的伺服器，無需記憶多個密碼。
JumpServer
開源堡壘機產品，支持 SSH/RDP/VNC 等多種協議，提供 Web 界面管理，支持雙因素認證（MFA）。
JumpServer 快速安裝（官方一鍵腳本）
# JumpServer 官方一鍵安裝腳本 [root@localhost ~]# curl -sSL https://resource.fit2cloud.com/jumpserver/jumpserver/releases/latest/download/quick_start.sh | bash # 安裝完成後訪問：http://伺服器IP # 默認帳號：admin # 默認密碼：ChangeMe（首次登入需修改） # 使用 Docker 安裝（推薦用於測試環境） [root@localhost ~]# docker run -d \ --name jms_all \ -v /opt/jumpserver/data:/opt/jumpserver/data \ -p 80:80 -p 2222:2222 \ --restart=always \ jumpserver/jms_all:latest

JumpServer 安全策略配置

JumpServer 安裝完成後，需要進行基本的安全配置才能投入使用。

配置步驟操作說明安全意義
1. 修改默認密碼首次登入強制修改 admin 密碼防止默認憑證被利用
2. 開啟 MFA 雙因素認證系統設置 > 開啟 MFA即使密碼洩漏也無法登入
3. 創建資產（伺服器）資產管理 > 資產列表 > 創建資產集中管理所有被管伺服器
4. 創建系統用戶配置伺服器的登入帳號（如 deploy）運維人員不需要知道伺服器密碼
5. 配置授權規則設定哪些用戶可以訪問哪些資產最小化權限原則
6. 查看操作審計審計 > 會話在線 / 歷史會話可回放所有操作錄影

OpenVPN 原理與搭建實戰

OpenVPN 是一個開源的 VPN 解決方案，使用 SSL/TLS 協議加密流量。企業常用 OpenVPN 讓員工在家安全地訪問公司內網。對於資安工程師，理解 VPN 原理也有助於理解 VPN 繞過和隧道技術。
