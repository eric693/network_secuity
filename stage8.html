社會工程學概論與攻擊框架

社會工程學（Social Engineering）是利用人類的心理弱點、信任機制和行為習慣，誘使目標個人自願提供信息或執行特定動作的攻擊藝術。它繞過技術防禦，直接攻擊「人」這個最脆弱的環節。

為什麼社工攻擊如此有效？

根據 Verizon DBIR 報告，超過 80% 的資料洩露涉及人為因素——社工攻擊、憑證竊取或釣魚郵件。組織可以修補系統漏洞，但無法「修補」人的心理。即使企業有最先進的技術防禦，一封精心設計的釣魚郵件仍可能讓員工點擊並交出憑證。

社工攻擊完整框架（SEEF）

PHASE 1
情報收集（Intelligence Gathering）
通過 OSINT 工具（LinkedIn、GitHub、WHOIS、Shodan）收集目標組織架構、員工姓名職位、業務流程、技術棧、合作夥伴等信息。信息越豐富，後續攻擊越有針對性。
PHASE 2
目標選定（Target Identification）
根據目標確定最佳攻擊對象：財務人員（轉帳詐騙）、IT 人員（密碼重置）、高管助理（魚叉釣魚）、前台接待（實體社工）。選對目標事半功倍。
PHASE 3
借口設計（Pretexting）
構建可信的虛假身份和情景：假扮 IT 支援人員、審計師、合作廠商、高管等。設計具體的對話腳本，包括如何建立信任、如何引導對話，以及應對常見疑問的回應。
PHASE 4
攻擊執行（Attack Execution）
通過選定的攻擊向量執行：釣魚郵件、Vishing（語音釣魚）、Smishing（SMS 釣魚）、實體滲透、USB 丟棄。在執行過程中靈活運用心理影響技術。
PHASE 5
目標達成（Objective Achievement）
獲得憑證（用於內網滲透）、惡意代碼執行（初始立足點）、機密信息洩露、財務欺詐授權。社工演練中需完整記錄哪些員工被成功社工，以生成報告。
PHASE 6
報告與培訓（Report & Training）
整理演練數據（點擊率、憑證提交率、通報率），分析薄弱環節，提供具體的意識培訓建議和技術防禦措施。社工演練的最終目的是提升組織防禦能力。
授權與法律邊界

社工演練在許多國家涉及詐騙罪、偽造文件、不法侵入等法律風險。必須取得明確的書面授權，包括：（1）客戶聲明書（Statement of Work）明確列出允許的攻擊向量；（2）緊急中止程序（如員工向真實IT部門求助時的通知機制）；（3）明確排除某些敏感人員（如已知有心理健康問題的員工）。沒有授權的任何社工行為均屬犯罪。

影響力原理：六大心理觸發機制

Robert Cialdini 在《影響力》中總結了六大影響力原理，這些心理機制被社工攻擊者廣泛利用。理解這些原理是設計有效社工攻擊和防禦培訓的基礎。

影響力原理心理機制社工攻擊應用典型話術
互惠（Reciprocity）收到恩惠後產生回報義務感先提供「免費幫助」，再要求敏感信息「我剛幫你解決了郵箱問題，你能確認一下帳號信息讓我登記嗎？」
承諾一致（Commitment）人傾向於保持言行與已有承諾一致先獲得小承諾，再逐步升級要求「你之前同意配合我們的安全審查，所以現在需要你確認一下密碼。」
社會認同（Social Proof）不確定時跟隨大多數人的行為聲稱其他同事已經配合「你們部門的同事都已經完成驗證了，就差你一個。」
權威（Authority）服從權威人物的指令假冒高管、IT部門、執法機構「我是 CEO 辦公室的，張總要求你立即處理這個緊急事項。」
稀缺（Scarcity）稀缺資源更具吸引力，時間壓力促使倉促決策製造緊迫感，阻止目標理性思考「你的帳號將在 15 分鐘後被鎖定，請立即點擊鏈接驗證身份。」
喜好（Liking）更容易被自己喜歡或信任的人影響建立個人連結感、共同點、讚美目標「我知道你是工程部門最厲害的，所以特地來請教你這個技術問題。」
組合使用心理觸發器

最有效的社工攻擊通常同時使用多種觸發機制。例如一封釣魚郵件可以同時使用：權威（假冒 CEO 或 IT 部門）+ 稀缺/緊迫（「帳號將在 24 小時內停用」）+ 社會認同（「所有員工都已完成此更新」）。三重心理壓力使目標難以保持理性分析。在撰寫釣魚郵件時，識別並刻意融入至少 2-3 個觸發機制。

目標偵察：OSINT 人員情報收集

在社工攻擊前，針對目標組織和個人的深入情報收集（OSINT）是提高攻擊成功率的關鍵。收集的信息越詳細，構建的借口越可信。

LinkedIn — 組織架構與人員
枚舉員工姓名、職位、部門、直屬關係。通過「In Common」找到目標的共同聯繫。查看職位描述推斷技術棧（「使用 Salesforce / SAP / AWS」）。舊版 LinkedIn API 可批量導出員工數據。
GitHub — 技術信息洩露
搜索公司域名郵件（@company.com）找到開發人員帳號，查看其公開倉庫中是否有硬編碼憑證、內部 API 端點、基礎設施信息。site:github.com "company.com" password
Hunter.io / Clearbit — 郵件格式
自動偵測公司郵件格式（firstname.lastname@company.com 或 f.lastname@company.com）並驗證郵件有效性。一旦知道格式，可以構建任意員工的郵件地址用於魚叉釣魚。
Maltego — 可視化關聯分析
將 OSINT 數據視覺化為關係圖：域名 → DNS → IP → 組織 → 員工 → 社交媒體帳號。Transform 自動執行多個 OSINT 查詢，適合複雜目標的全面偵察。

借口構建（Pretexting）技術

Pretexting 是創建虛假情景和身份，為社工攻擊提供「合理」的框架。一個好的借口需要：與目標業務環境高度相關、利用已收集的 OSINT 信息增加可信度、設計完整的對話流程，包括應對質疑的回應。

01
IT 技術支援 — 最常用的借口。假冒內部 IT 人員或外包 IT 公司，以「系統升級」「帳號安全驗證」「VPN 配置」等理由索取憑證或誘使安裝軟件。成功率高，因為員工習慣於服從 IT 指令。
02
高管指令（BEC — Business Email Compromise） — 假冒 CEO/CFO 向財務人員發出緊急轉帳指令。2023 年 FBI 數據顯示，BEC 詐騙年損失超過 27 億美元，是最具財務破壞力的社工攻擊類型。
03
供應商 / 合作夥伴 — 假冒目標組織已知的供應商（如辦公室設備維修、雲服務提供商）。事先通過 OSINT 了解真實供應商名稱可大幅提升可信度。
04
新員工 / 人力資源 — 假扮新入職員工（需要網絡訪問幫助），或假冒 HR 部門發送「年度調查問卷」「薪資調整確認」等郵件。目標通常對新員工更有耐心。
05
監管機構 / 審計師 — 假冒稅務局、行業監管機構、第三方審計師。「法規合規」話題製造強烈的緊迫感和服從壓力，特別對法務、財務部門員工有效。
06
媒體記者 / 研究人員 — 假冒記者以採訪為名索取敏感信息。目標可能在未被告知什麼可以公開的情況下透露過多業務細節，甚至提供內部文件「佐證」。
演練中的邊界線

即使在授權的演練中，以下行為通常被禁止或需要額外授權：向員工施加造成心理創傷的壓力（如長時間恐嚇）、針對已知脆弱員工（新入職者、有已知壓力的員工）、涉及真實財務轉帳（即使是演練，也應在完成後立即通知財務部門）、收集超出演練目的的個人資料。始終在授權範圍內操作。

小測驗
一個釣魚郵件的主旨是「【緊急】您的企業帳號將於今日 17:00 停用，請在截止時間前點擊連結完成驗證——所有部門員工已完成此步驟」。這個郵件同時利用了哪兩種 Cialdini 影響力原理？
A互惠（Reciprocity）+ 喜好（Liking）
B承諾一致（Commitment）+ 權威（Authority）
C稀缺/緊迫（Scarcity）+ 社會認同（Social Proof）
D喜好（Liking）+ 互惠（Reciprocity）
正確！「今日 17:00 停用」製造了時間壓力，觸發稀缺/緊迫原理，阻止目標仔細思考；「所有部門員工已完成此步驟」觸發社會認同原理，讓目標感到自己是唯一未完成的人，產生從眾壓力。這是最常見的釣魚郵件組合，研究表明同時使用兩種以上觸發機制可使點擊率提升 2-3 倍。

釣魚郵件：設計與投遞技術

釣魚郵件（Phishing Email）是最廣泛使用的社工攻擊向量。一封有效的釣魚郵件需要在技術（繞過郵件過濾）和心理（觸發目標行動）兩個層面都達到高水準。

釣魚郵件解剖

寄件人偽造（From Spoofing）
目標：讓郵件看起來來自可信源。技術手段：1. 使用同形字攻擊（rnicrosoft.com vs microsoft.com）; 2. 顯示名稱偽造（顯示「IT Support」但實際地址是 hacker@gmail.com）; 3. 合法域名的子域（support@mail.microsoft-security.com）。
主旨行設計
關鍵詞觸發行動：緊急（URGENT）、安全（Security Alert）、行動（Action Required）、獨家（Your account）。A/B 測試不同主旨，使用 GoPhish 追蹤打開率。避免大量感嘆號和全大寫（觸發垃圾郵件過濾）。
正文可信度
使用目標組織的真實 Logo 和品牌色彩（從官網直接引用 img src URL 更真實）；匹配目標的企業語氣；加入個人化信息（真實姓名、部門、職位）；避免語法錯誤（觸發懷疑）。
行動號召（CTA）
釣魚鏈接設計：使用短網址（bit.ly）或合法域前置（CDN 重定向）隱藏真實 URL；或惡意附件（.docx 含宏、.pdf 含 JS、.zip 含 .lnk 或 .iso）；QR Code 釣魚（繞過部分郵件過濾）。
釣魚郵件模板結構（高效模式）
主旨：【安全通知】您的企業 Microsoft 365 帳號需要立即驗證

─────────────────────────────────────
[Microsoft Logo]
─────────────────────────────────────

親愛的 {{FirstName}}，

我們檢測到您的企業帳號（{{Email}}）近期有以下異常登入活動：

  登入時間：{{CurrentDate}} 03:42 AM
  地理位置：俄羅斯 莫斯科
  裝置：未知 Windows 設備

                [ 立即驗證帳號 ]
      https://account-verify.{{LegitLookingDomain}}.com

─────────────────────────────────────

# 結構分析：

# 2. 具體威脅（真實時間+陌生地點）→ 製造恐懼
# 3. 時間壓力（24小時內）→ 稀缺原理
# 4. 退出選項（忽略）→ 降低防禦心理
# 5. 權威簽名（Microsoft 安全團隊）→ 權威原理

魚叉式釣魚（Spear Phishing）

魚叉式釣魚（Spear Phishing）是針對特定個人或小型群體的高度客製化釣魚攻擊，區別於廣撒網的大規模釣魚。它需要更多前期 OSINT 準備，但成功率遠高於一般釣魚。

類型目標特點典型案例
Spear Phishing特定個人（1-10人）高度個人化，基於詳細 OSINT；成功率 50%+；需要大量準備時間釣魚針對 CFO，使用其直屬上司的姓名和最近業務事件
Whaling（捕鯨）高管（CEO/CFO/CTO）最高價值目標；郵件格式更正式；後果最嚴重（BEC 詐騙、數據洩露）假冒 CEO 要求 CFO 向「收購目標」匯款
Clone Phishing已收到合法郵件的目標克隆目標此前收到的真實郵件（快遞通知/會議邀請），替換鏈接或附件克隆 DHL 快遞通知，替換追蹤鏈接為釣魚頁面
Vishing（語音）電話通話目標實時對話，可靈活應對；不受郵件過濾影響；結合 AI 聲音克隆技術效果更強假冒銀行客服，誘使客戶提供 OTP

GoPhish 框架：自動化釣魚演練

GoPhish 是一個開源的釣魚演練平台，專為滲透測試人員和安全團隊設計，提供釣魚活動管理、郵件模板、追蹤分析等完整功能，是大規模組織釣魚演練的標準工具。

GoPhish 部署與配置

釣魚頁面製作：Evilginx / GoPhish 落地頁

釣魚頁面（Landing Page / Phishing Page）是誘使目標輸入憑證的關鍵。現代釣魚頁面不僅要外觀逼真，還需要能繞過 MFA——這是 Evilginx 等中間人釣魚工具的核心能力。

技術原理能否繞過 MFA工具
靜態克隆頁面複製目標登入頁面的 HTML/CSS，捕獲用戶輸入的憑證否（只能竊取密碼，無法處理 OTP）GoPhish / HTTrack / wget
反向代理釣魚作為中間人在用戶和真實服務器之間，實時轉發並捕獲所有請求（含 Session Cookie）是（劫持完整驗證後的 Session Cookie）Evilginx3 / Modlishka
OAuth/SSO 釣魚偽造 OAuth 授權頁，誘使用戶授予攻擊者控制的應用訪問權限是（獲得 OAuth 令牌，有效期長）自建 OAuth App + GoPhish
Browser-in-the-Browser在頁面內用 CSS/JS 模擬真實的瀏覽器彈窗（如 Google 登入視窗），URL 欄顯示合法地址否（仍是偽造頁面）BitB toolkit
Evilginx3 — 核心優勢

Evilginx3 使用反向代理架構，讓目標直接與真實的 Microsoft / Google 服務器通訊，因此：（1）SSL 憑證是真實的（鎖頭圖示正常）；（2）真實的 MFA 流程正常完成；（3）Evilginx 在中間捕獲驗證後的 Session Cookie，攻擊者可直接用此 Cookie 繞過 MFA 訪問帳號。這就是為什麼即使啟用了 SMS OTP 或 TOTP，針對 Evilginx 攻擊也無法防禦——唯一有效防禦是 FIDO2 硬體金鑰（Origin 綁定，中間人無法重放）。

Vishing（語音釣魚）腳本設計

Vishing（Voice Phishing）通過電話進行社工攻擊。相比郵件，電話對話是實時的，需要即時應對目標反應，但也因此更有說服力——真人聲音比文字更難讓人拒絕。

Vishing 腳本範例（IT 技術支援借口）
【場景】：假冒公司 IT 服務台，向員工索取 VPN 憑證進行「安全更新」 【前置工作】：從 LinkedIn 確認目標姓名和部門；從官網找到 IT 服務台的真實電話號碼 （以便目標「確認」時說「我們的號碼是 XXXX，這正是我們的號碼」） ──────────────────────────────── 攻擊者：「你好，請問是行政部的陳小姐嗎？我是 IT 服務台的阿明， 工號 T2047。我們的系統顯示您的電腦有一個緊急安全更新需要處理， 是關於 VPN 客戶端的漏洞補丁。」 目標：「喔，是嗎？我現在電腦用起來是正常的。」 攻擊者：「是的，這個漏洞是靜默的，用起來表面正常，但後台可能已經被 攻擊者入侵。我們公司昨天有兩台機器中招了。您方便讓我確認一下 您的帳號信息嗎？只需要幾分鐘。」 目標：「好的，我的帳號是 cchen。」 攻擊者：「好的，謝謝。現在我需要您配合我做個驗證：我會在系統後台重置 您的 VPN 密碼，新密碼會發到您手機。您收到後，請直接告訴我那個 6 位數字，這樣我才能確認您的賬號已經更新成功。」 ──────────────────────────────── # ─── Vishing 關鍵技術要點 ─── # 1. 電話號碼欺騙（Caller ID Spoofing） # 使用 SIP 服務（如 SpoofCard / Twilio）偽造來電顯示為公司內線號碼 # 工具：antispoofing.com, twilio.com（API 發起語音通話並設定發件人號碼） # 2. 說話節奏控制 # - 語速適中（過快顯得緊張） # - 適當使用停頓製造自信感 # - 使用目標的名字（建立個人聯繫） # 3. 應對質疑的回應 # 目標：「我能先確認一下你的工號嗎？」 # 回應：「當然，我的工號是 T2047，您也可以撥打公司總機轉 IT 服務台確認， # 但如果要等轉接的話，您的帳號更新可能來不及在今天 5 點前完成。」 # （權威 + 稀缺：給予核實機會但同時製造時間壓力）

尾隨進入（Tailgating）與實體滲透

實體社工（Physical Social Engineering）攻擊的是物理安全控制，包括門禁系統、訪客登記程序和員工安全意識。尾隨進入（Tailgating）是最常見的實體攻擊手段，成功率驚人地高。

01
尾隨進入（Tailgating / Piggybacking） — 緊跟已刷卡進入的員工進入門禁區域，不使用自己的憑證。研究顯示，超過 70% 的員工會為「看起來友善」的陌生人開門。常見偽裝：雙手提著文件箱（顯得騰不出手刷卡）、穿工程師背心或送餐員制服。
02
偽造訪客身份 — 以外部人員身份（IT 設備維修、辦公室植物護理、快遞員）進入。前台可能不嚴格驗證預約，尤其當聲稱是「緊急維修」或「設備保養」時。準備看起來專業的工具包和名片。
03
肩窺（Shoulder Surfing） — 在公共區域（咖啡廳、機場、開放辦公室）偷看目標輸入密碼或查看機密屏幕。高倍率攝影鏡頭可以在更遠距離進行。實體紅隊演練中需明確記錄此類機密數據暴露情況。
04
桌面信息收集（Dumpster Diving / Desk Inspection） — 在無人看管時快速拍攝桌上的便條紙（常記有密碼）、組織圖、印出的文件。翻查垃圾桶（Dumpster Diving）可找到含敏感信息的廢棄文件。
05
RFID 複製 — 使用 Proxmark3 等工具在近距離（30cm 內）讀取目標的門禁卡並複製，製作一張功能相同的卡。低頻 125kHz 卡（常見舊款門禁）可輕鬆複製；13.56MHz MIFARE 的加密變體更難但仍有已知漏洞。

USB 丟棄攻擊與惡意設備

USB 丟棄（USB Drop）攻擊利用人類的好奇心——在停車場或辦公大廳故意遺留標有吸引標籤（如「薪資表 2024」「CEO 私人文件」）的 USB 隨身碟。研究顯示，約 45% 的人會插入撿到的 USB。

普通 USB + AutoRun
早期技術，依賴 Windows AutoRun 功能自動執行。Windows 7+ 默認禁用 AutoRun，但仍可用於誘使用戶手動點擊惡意文件（.lnk 偽裝成文件夾）。現代防毒軟件通常可偵測。
USB Rubber Ducky
偽裝成普通 USB 隨身碟的 HID（鍵盤）設備，插入後自動模擬鍵盤輸入，在幾秒內執行預設腳本（開啟 PowerShell、下載並執行 Payload、新增用戶帳號）。速度極快，人眼難以察覺。
O.MG Cable
偽裝成普通 USB 充電線的攻擊設備，內置 WiFi 模組和鍵盤模擬器。攻擊者可遠端通過 WiFi 向連接的電腦發送惡意鍵盤輸入，或在特定條件下觸發攻擊。視覺上與正常充電線完全相同。
Bash Bunny
多模式攻擊設備（可切換 HID/Ethernet/Mass Storage）。可執行複雜的後滲透腳本：自動轉儲 Windows 憑證（QuickCreds 模組）、安裝持久化後門、導出瀏覽器存儲的密碼。

深偽技術（Deepfake）與 AI 社工

AI 技術的發展正在革命性地改變社工攻擊的能力。語音克隆、視頻深偽和 AI 生成釣魚郵件大幅降低了高度可信攻擊的門檻。

AI 技術攻擊應用真實案例防禦難點
語音克隆（Voice Cloning）僅需 3-10 秒語音樣本（來自 YouTube / LinkedIn 視頻）即可克隆 CEO 聲音，用於 Vishing；工具：ElevenLabs、RVC2020 年，UAE 銀行行長聲音被克隆，誘使分行匯款 3500 萬美元電話語音無法可靠驗證身份；「緊急情況」使人跳過驗證
AI 釣魚郵件生成使用 LLM 自動生成個人化、無語法錯誤的釣魚郵件；可根據目標 LinkedIn 資料自動客製化WormGPT、FraudGPT 等無審查 LLM 在暗網出售AI 生成郵件比人工撰寫更難通過文風識別
視頻深偽（Video Deepfake）在視頻通話中實時替換臉部，假冒高管進行視頻會議指令；工具：DeepFaceLive、FaceFusion2024 年，香港跨國公司員工在視頻會議被 Deepfake CFO 欺騙，匯款 2 億港元視頻通話的真實感大幅降低員工警覺性
AI 個人化釣魚（Spear Phishing at Scale）將 LLM 與 OSINT 工具結合，自動為大量目標生成高度個人化的魚叉釣魚郵件，規模化精準攻擊研究顯示 AI 生成的個人化釣魚郵件點擊率比通用模板高 40%傳統基於規則的過濾無法識別高品質 AI 生成內容

域名策略：釣魚域選購與配置

釣魚基礎設施（Phishing Infrastructure）的設計決定了攻擊的成功率和持久性。域名選擇是第一步，也是整個釣魚行動的基礎。

釣魚域名類型與選購策略
# ─── 釣魚域名技術（視覺混淆）───

# 1. Typosquatting（拼寫錯誤）
# 目標：microsoft.com
microsofy.com          # 末尾字母替換
micros0ft.com          # 數字替換字母
micosoft.com           # 缺少字母

# 2. 同形字攻擊（Homograph / IDN Homograph）
# 使用 Unicode 字符替換拉丁字母，視覺上幾乎相同
mіcrosoft.com          # i → 西里爾文 і（U+0456）
rnicrosoft.com         # m → rn（兩個字母組合）

# 3. 添加關鍵詞（Combosquatting）
microsoft-secure.com

microsoft-support.co

# 4. 相似 TLD

microsoft.com.cn       # 添加地區後綴

# 全新域名的郵件更易被標記為垃圾郵件
# 策略1：購買已有一定年齡（1-2年）的已過期域名
# 查詢工具：expireddomains.net、SpamHaus Checker

# 策略2：預熱新域名（在釣魚演練前 2-4 週）：
# - 設置 DNS MX 記錄、SPF/DKIM/DMARC
# - 從此域名發送少量非釣魚郵件（新聞通訊、測試郵件）
# - 建立一個看起來合法的網站

# ─── 釣魚域名基礎 DNS 配置 ───
# 在 Cloudflare / Route53 配置以下 DNS 記錄：
# A 記錄：phish.example.com → [VPS IP]（GoPhish 或 Evilginx 服務器）
# MX 記錄：example.com → mail.example.com（自建郵件服務器）
# TXT 記錄（SPF）：v=spf1 ip4:[VPS IP] -all
# TXT 記錄（DMARC）：v=DMARC1; p=none; rua=mailto:dmarc@example.com
# CNAME（DKIM）：mail._domainkey → [DKIM 公鑰記錄]
# SSL 憑證（Let's Encrypt）：
certbot certonly --standalone -d phish.example.com --agree-tos -m admin@example.com

釣魚郵件繞過 SPF / DKIM / DMARC

現代郵件安全系統通過 SPF、DKIM、DMARC 三重機制驗證發件人身份。理解這些機制及其繞過方法，是確保釣魚郵件能夠送達目標收件箱的關鍵。

機制驗證內容常見弱點繞過策略
SPF發件 IP 是否在域名授權的 IP 列表中（DNS TXT 記錄）許多組織使用 ~all（軟失敗）而非 -all（硬拒絕）；Forwarding 場景下 SPF 必然失敗使用自有域名正確配置 SPF；利用 Open Relay 轉發；購買 SPF 配置鬆散的域名
DKIM郵件內容的數字簽名，使用域名私鑰簽名；接收方用 DNS 公鑰驗證舊版 DKIM 使用 512/1024-bit 密鑰（已可被破解）；DKIM 只驗證 Header-From，Body-From 可不同對自有域名設定 DKIM；利用第三方 ESP（Mailgun/SES）的 DKIM；DKIM Replay Attack
DMARCSPF 或 DKIM 是否與 From 域名對齊；定義驗證失敗後的處理策略（none/quarantine/reject）大量域名使用 p=none（僅報告，不攔截）；DMARC 子域名策略（sp=）常被忽略針對 p=none 的域名直接發送；使用相似域名繞過 DMARC 對齊
郵件安全配置驗證工具
# ─── 查詢目標域名的 SPF / DKIM / DMARC 配置 ───
# 了解目標的配置弱點，決定攻擊策略

# SPF 查詢
nslookup -type=TXT targetcorp.com | grep spf
# 結果示例：v=spf1 include:spf.protection.outlook.com ~all
# ~all = 軟失敗（很多郵件服務器仍會投遞，只是標記）
# -all = 硬失敗（嚴格拒絕）

# DMARC 查詢
nslookup -type=TXT _dmarc.targetcorp.com
# 結果示例：v=DMARC1; p=none; rua=mailto:dmarc@targetcorp.com
# p=none = 只報告，不攔截（直接發送即可）

# DKIM 選擇器查詢（需要知道選擇器名稱，常見：mail / google / s1 / s2）
nslookup -type=TXT mail._domainkey.targetcorp.com

# ─── 線上工具（更快速）───

# Mail Tester: https://www.mail-tester.com（測試你的發件信譽分數）
# dmarcian: https://dmarcian.com/dmarc-inspector/

# ─── 測試你的釣魚郵件能否進入收件箱 ───
# 使用自己控制的測試帳號（Gmail/Outlook）接收並查看郵件頭部
# Gmail：查看「原始郵件」→ 確認 DKIM-Signature / ARC-Authentication-Results
# 確認：dmarc=pass、spf=pass、dkim=pass

Payload 投遞：宏 / LNK / ISO

Payload 投遞（Payload Delivery）是通過釣魚郵件在目標機器上執行惡意代碼的關鍵環節。隨著 Microsoft 2022 年默認禁用網際網路下載文件的 VBA 宏，攻擊者不斷發展新的投遞技術。

01
VBA 宏（Office Macro） — 最傳統但仍有效的方式。Microsoft 365 現在對從網際網路下載的文件默認禁用宏，但在組織內部共享、或通過 LAN/UNC 路徑分發時宏仍可執行。規避：嵌入 Excel 4.0 XLM 宏（舊格式，不同的限制規則）。
02
LNK 文件（Windows 捷徑） — 偽裝成文件夾或文件的 .lnk 捷徑，點擊時執行自訂命令（如 PowerShell 下載並執行 Payload）。通常打包在 ZIP 中附件發送，因為 .lnk 直接發送可能被過濾。Qbot / Emotet 等惡意軟件廣泛使用此技術。
03
ISO / IMG 掛載文件 — 將 LNK 或可執行文件打包在 ISO 光盤映像中。Windows 10+ 雙擊 ISO 自動掛載，ISO 內的文件不帶 Mark-of-the-Web（MOTW）標記，因此不受「阻止網際網路下載文件的宏」限制。目前仍有效，但 Windows 11 2022 更新後對 ISO 也添加了 MOTW 傳播。
04
HTML Smuggling — 將 Payload 以 Base64 編碼嵌入 HTML 附件中，通過 JavaScript Blob 在受害者瀏覽器端重新組裝並自動觸發下載。Payload 不在郵件伺服器或 HTTP 傳輸中以可識別格式存在，繞過大多數郵件掃描閘道。
05
OneNote 嵌入（.one 文件） — 2022-2023 年流行。Microsoft OneNote 文件可嵌入任意文件，點擊頁面內的「按鈕圖片」即觸發執行。Microsoft 在 2023 年 3 月更新後添加了警告，但仍被使用。
小測驗
在授權的釣魚演練中，你使用 Evilginx3 設置了 Microsoft 365 釣魚頁面。目標員工完整地完成了 M365 登入流程，包括 SMS OTP 驗證。你在 Evilginx3 中捕獲了什麼，可以讓你在不知道密碼和 OTP 的情況下訪問其帳號？
A用戶的 NTLM Hash（可用於 Pass-the-Hash）
B用戶的明文密碼（Evilginx 解密了 HTTPS 流量）
C完整認證後的 Session Cookie（包含 AccessToken 和 RefreshToken）
DSAML Assertion（SAML 令牌，有效期極短）
正確！Evilginx3 的核心能力是捕獲完整認證後的 Session Cookie。當用戶完成包括 MFA 在內的所有驗證步驟後，Microsoft 服務器會發送包含訪問令牌的 Session Cookie 到瀏覽器。Evilginx 作為中間人攔截了這個 Cookie，攻擊者可以直接將這個 Cookie 注入自己的瀏覽器，「繼承」目標的已認證 Session，完全繞過密碼和 MFA 驗證。這就是為什麼 FIDO2 硬體金鑰是唯一有效防禦——因為 FIDO2 認證與 Origin（域名）綁定，即使攻擊者截獲了請求，也無法在不同域名下重放。

社工演練報告：數據與建議

釣魚演練的最終交付物是報告，它需要將技術發現轉化為管理層可理解的業務風險，並提供可操作的改進建議。

關鍵指標（KPI）
郵件投遞率（Delivery Rate）、郵件開啟率（Open Rate）、釣魚鏈接點擊率（Click-Through Rate，CTR）、憑證提交率（Credential Submission Rate）、安全事件通報率（Report Rate）——最後一項是最重要的正面指標。
分群分析
按部門、職級、地區分析釣魚成功率。高風險部門（財務、IT、高管助理）通常需要額外培訓。識別重複點擊者（Repeat Clickers）進行個人化介入。
攻擊時間窗
分析哪個時間段點擊率最高（通常是週一早上和週五下午）。工作開始前的郵件或下班前的郵件往往成功率更高，因為員工匆忙處理。
對比基準
將結果與行業基準對比（如 KnowBe4 的行業 Phish-Prone Percentage 報告）；與組織歷史數據對比，展示趨勢改善或惡化；設定下次演練的改進目標。
GoPhish 演練數據導出與分析
# ─── GoPhish API 導出演練數據 ───
# 獲取 API 密鑰：管理界面 → Account Settings → API Key

# 列出所有活動
curl -k "https://127.0.0.1:3333/api/campaigns/?api_key=YOUR_API_KEY"

# 獲取特定活動的詳細結果
curl -k "https://127.0.0.1:3333/api/campaigns/1/results?api_key=YOUR_API_KEY" \
  | python3 -m json.tool > results.json

# ─── Python 分析腳本（計算各指標）───
import json

with open('results.json') as f:
    data = json.load(f)

results = data['results']
total = len(results)
opened = sum(1 for r in results if r['status'] != 'Email Sent')
clicked = sum(1 for r in results if r['status'] in ['Clicked Link', 'Submitted Data'])
submitted = sum(1 for r in results if r['status'] == 'Submitted Data')
reported = sum(1 for r in results if r['status'] == 'Email Reported')

print(f"總目標：{total}")
print(f"郵件開啟率：{opened/total*100:.1f}%")
print(f"點擊率（CTR）：{clicked/total*100:.1f}%")
print(f"憑證提交率：{submitted/total*100:.1f}%")
print(f"可疑郵件通報率：{reported/total*100:.1f}%")

員工安全意識培訓設計

技術防禦永遠不能完全消除社工風險，持續的安全意識培訓（Security Awareness Training，SAT）是讓員工成為「人防火牆」的唯一途徑。有效的培訓需要改變行為，而不只是傳遞知識。

01
即時教育（Just-in-Time Training） — 最有效的方法：員工點擊釣魚鏈接後，立即在釣魚落地頁上顯示培訓內容（「你剛剛上當了！這是一次釣魚演練，以下是識別方法…」）。記憶在情緒觸動後最深刻。GoPhish 的 Landing Page 可直接配置此功能。
02
短小精悍的微培訓（Micro-learning） — 與其每年一次 2 小時的枯燥 PPT，不如每月 1 次 5 分鐘的互動模組效果更好。KnowBe4、Proofpoint SAT、SANS Securing The Human 等平台提供大量現成模組。
03
模擬演練頻率漸進 — 初始演練使用較容易識別的釣魚（讓員工建立信心）；隨後逐步增加難度（添加個人化信息、使用更可信的借口）；對重複點擊者進行一對一跟進培訓。研究顯示月度演練可使釣魚成功率在 12 個月內降低 70%+。
04
通報文化建設 — 比「不要點擊」更重要的是「點擊了要立即通報」。培訓重點應包括：如何使用「Report Phishing」按鈕（Outlook 插件 / Google Workspace 功能）；通報後的流程；確保員工不因通報釣魚郵件而受到懲罰（即使是真實郵件）。
05
正面激勵 — 避免「羞辱」點擊者的文化。公開表揚第一個通報特定釣魚郵件的員工；建立通報排行榜；每月通報最多的部門給予認可。將安全意識轉化為積極的組織文化而非恐懼文化。

技術防禦：郵件閘道 / MFA / FIDO2

社工攻擊的技術防禦需要多層次部署，每一層都針對攻擊鏈的不同環節。沒有任何單一技術能完全防止社工攻擊，但組合使用可大幅降低成功率。

防禦層次技術方案防禦效果局限性
郵件閘道Microsoft Defender for Office 365 / Proofpoint / Mimecast；安全連結（URL 重寫掃描）、安全附件（沙盒分析）、反欺騙（SPF/DKIM/DMARC 嚴格執行）高（攔截 95%+ 已知釣魚）無法識別新型釣魚域名；合法服務（Google Forms、SharePoint）的釣魚難以攔截
MFA（TOTP/SMS）時間型一次性密碼（TOTP）如 Google Authenticator；SMS OTP；推送通知（Microsoft Authenticator）中（防止密碼洩露後的帳號接管）無法防止 Evilginx/AiTM 中間人攻擊（Cookie 劫持繞過 MFA）；Push Bombing 攻擊
FIDO2 / 硬體金鑰YubiKey / Google Titan Key；Windows Hello for Business（TPM 綁定）；Passkeys（FIDO2 平台驗證器）極高（加密與 Origin 綁定，無法被中間人重放）部署成本高；用戶習慣改變；並非所有服務支援
條件訪問策略Entra ID（Azure AD）條件訪問：要求合規設備、指定 IP 範圍、風險登入自動阻擋或要求額外驗證高（即使 Cookie 被劫持，陌生設備/位置也會觸發阻擋）配置複雜；合法遠端工作場景可能被誤擋
郵件外部標籤在所有外部郵件主旨或頂部添加「[外部]」警示標籤，讓員工立即識別非內部郵件中（顯著提升員工警覺性）用戶可能習慣後忽視；不影響真實內部帳號被盜用的攻擊
